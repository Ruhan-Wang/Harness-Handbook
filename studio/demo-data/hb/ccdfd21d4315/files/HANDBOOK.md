This repo is a *harness* — a codebase whose behavior is best understood as a
lifecycle of stages and the state that flows between them. A handbook makes that
lifecycle explicit so a human (or an AI agent) can navigate it quickly.

## What a "handbook" is

Handbook Studio analyzes the code and produces, under `.handbook/` in this repo:

- **Call graph** — who calls whom (built from the AST, no LLM).
- **Skeleton** — the lifecycle *stages* and the *state registers* that carry data
  between them.
- **Stage + function docs** — plain-language explanations of each stage and the
  functions mapped into it, with `file:line` references.
- **`SKILL.md`** — a compact, agent-loadable summary so a coding agent can use
  the handbook as context.

## How to start a handbook

1. In **Handbook Studio**, select **Terminus 2** in the left sidebar.
2. (Recommended) Set the **scope** field to a focused subfolder (e.g. a single agent/package) — leave blank to scan the whole repo. A tight scope keeps
   generation fast and cheap.
3. Pick a **Quality** mode:
   - **Thorough** — full actor-critic (best quality, most LLM calls).
   - **Fast** — one critic, no revise rounds (good default).
   - **Minimal** — actor only, no critic (fastest, roughest).
4. Click **Generate full handbook**. Live progress streams in the Workspace.
5. When it finishes, open the **Handbook** or **Split view** tab to explore the
   call graph, stages, and state registers — and chat with the agent grounded in
   the generated handbook.

| 4 | Write docs | per-stage + per-function explanations | bulk |
