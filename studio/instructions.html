Usage guide

Handbook Studio Instructions

Handbook Studio turns a large codebase into navigable behavioral knowledge, then connects every explanation back to implementation evidence. Use this guide to explore a handbook and understand the code-evolution workflow.

Demo notice: This is a static demo. It only supports browsing the handbooks we have already generated and replaying saved Co-Edit history. It does not generate new handbooks or run live edits.

03 · Co-Edit

From behavioral request to synchronized knowledge

RequestDescribe the change
HandbookLocate behavior
PlanIdentify code sites
PatchReview changes
SyncRefresh knowledge

Begin with a behavioral intent rather than a guessed filename. Handbook Studio uses the handbook to identify relevant behavior units and implementation evidence, presents a focused plan, then keeps the handbook aligned with the resulting code change.

Review the proposed plan and diff before applying a change. The handbook narrows the search space, but human judgment remains important for permissions, safety boundaries, and behavior that spans multiple paths.

04 · Static demo

What is available in this deployment

  • The bundled Codex and terminus_2 handbooks are fully browsable.
  • Saved Co-Edit sessions can be replayed from Evolution History.
  • Live repository generation, code execution, and model-driven editing require the local Handbook Studio application.
  • Changes made while browsing the static demo are not written back to a repository.

使用指南

Handbook Studio 使用说明

Handbook Studio 将大型代码库转化为可导航的行为知识，并把每一项说明连接回实现证据。 本指南将帮助你探索 Handbook，并理解从修改需求到代码演进的完整流程。

演示说明：当前展示的是静态 Demo，仅支持查看我们已经生成的 Handbook，以及回放已保存的 Co-Edit 历史记录；暂不支持生成新的 Handbook 或执行实时代码修改。
快速开始 核心视图 协同编辑流程 演示版限制

01 · 快速开始

探索已生成的 Handbook

打开示例代码库

从左侧“最近代码库”中选择 Codex 或 terminus_2。

先建立系统级理解

从 Handbook 概览开始，再依次查看行为阶段和共享状态，最后进入具体函数。

沿着证据进入代码

使用 Handbook ⇄ 代码，同时查看行为说明及其对应的源码位置。

回放一次修改

打开演进历史或协同编辑，查看一项需求如何转化为计划、补丁和同步后的 Handbook。

03 · 协同编辑

从行为需求到知识同步

需求描述修改目标
Handbook定位相关行为
计划确定代码位置
补丁审查代码变更
同步更新知识

从行为意图出发，而不是猜测文件名。Handbook Studio 根据 Handbook 定位相关行为单元与实现证据， 生成聚焦的修改计划，并在代码变化后保持 Handbook 与实现同步。

应用修改前，请审查计划与代码差异。Handbook 可以缩小搜索范围，但权限、安全边界以及跨路径行为仍需要人工判断。

04 · 静态演示

当前部署支持的功能

  • 可以完整浏览内置的 Codex 和 terminus_2 Handbook。
  • 可以从演进历史中回放已保存的协同编辑会话。
  • 实时生成代码库 Handbook、执行代码和模型驱动编辑，需要在本地运行完整的 Handbook Studio。
  • 在静态演示中进行的浏览操作不会写回代码库。
